# Phase 1 Audit

## 1. Stack and Architecture

### Audience and framing

This audit is written for the engineering team rebuilding the product as a clean multi-tenant SaaS in a separate repository. **They will not read this codebase.** This document is the spec. Sections 2–4 catalogue what's here so the rebuild knows what behaviour to preserve and what landmines to avoid; sections 5–7 give the rebuild team a starting roadmap.

### One-line summary

A single-tenant TypeScript monorepo: Express + tRPC backend, React 19 + Vite frontend, MySQL via Drizzle ORM, served as one Node process. Generated by Manus, deployed to Manus, depends on several Manus-platform proxy services (storage, OAuth, email, LLM, maps).

### Languages and runtime

- **TypeScript 5.9.3** end-to-end (`tsconfig.json`). `strict: true`, `module: ESNext`, `moduleResolution: bundler`, `noEmit: true` (Vite/esbuild emit). Path aliases: `@/* → client/src/*`, `@shared/* → shared/*`.
- **Node** (version unpinned — no `engines` field, no `.nvmrc`, no `.node-version`). Server is bundled with esbuild, dev runs through `tsx watch`.
- ES modules throughout (`"type": "module"` in `package.json`).

### Backend

- **Express 4.21.2** as the HTTP server (`server/_core/index.ts`). Body parser limits set to **50 MB** for JSON and urlencoded — large because client file uploads are sent as base64 in tRPC payloads.
- **tRPC 11.6.0** for the API (`@trpc/server`, `@trpc/client`, `@trpc/react-query`). Mounted at `/api/trpc` via `createExpressMiddleware`. The whole API is one router (`server/routers.ts`, ~2,200 lines, 27 sub-routers).
- **superjson 1.13.3** as tRPC transformer (handles Date/Map/Set).
- **zod 4.1.12** for input validation on every procedure.
- **jose 6.1.0** for JWT signing/verification (HS256). Two parallel JWT systems:
  - `server/_core/jwt.ts` — admin staff session (`bm_admin_session` cookie).
  - `server/_core/sdk.ts` — Manus OAuth user session (`app_session_id` cookie).
- **bcryptjs 3.0.3** for the single shared admin password hash. Cost 12.
- **multer 2.1.1** for the one Express upload route (`POST /api/boq/upload`).
- **cookie 1.0.2** for parsing cookies on raw `req.headers.cookie` (the codebase does not use `cookie-parser`).
- **nanoid 5.1.5** for client portal tokens and storage key suffixes.
- **axios 1.12.0** — used only by the Manus OAuth SDK (`server/_core/sdk.ts`). All other HTTP calls use `fetch`.
- **dotenv 17.2.2** loaded via `import "dotenv/config"` at the top of `server/_core/index.ts`.

### Frontend

- **React 19.2.1** + **react-dom 19.2.1**.
- **Vite 7.1.7** (`vite.config.ts`). Root is `client/`, public dir `client/public/`, build output `dist/public/`. envDir is the project root, so server and client share `.env`. Path aliases mirror tsconfig.
- **wouter 3.3.5** for routing — patched via `patches/wouter@3.7.1.patch` (pnpm.patchedDependencies). All app routes defined in `client/src/App.tsx`.
- **@tanstack/react-query 5.90.2** as the data layer; tRPC's React adapter wires it.
- **Tailwind CSS 4.1.14** via the **`@tailwindcss/vite`** plugin (no PostCSS config). Stylesheet: `client/src/index.css`. Custom oklch palette + `tw-animate-css`.
- **shadcn/ui** (style preset `new-york`, base color `neutral`, CSS variables on) — components live in `client/src/components/ui/` (~50 primitives wrapping Radix). Config in `components.json`.
- **Radix UI** primitives across the board (accordion, dialog, dropdown, popover, select, tabs, tooltip, etc.).
- **lucide-react 0.453.0** icons.
- **sonner 2.0.7** for toasts.
- **react-hook-form 7.64.0** + **@hookform/resolvers 5.2.2** + **zod** are in `package.json`, but **the largest forms (e.g. `AdminProjectForm.tsx`, 960 lines) are vanilla `useState`** — the form library is installed but not used on the main flows. shadcn's `form.tsx` wires it for components that opt in.
- **framer-motion 12.23.22**, **embla-carousel-react**, **recharts**, **react-day-picker**, **react-resizable-panels**, **input-otp**, **vaul**, **cmdk**, **next-themes**, **streamdown**, **xlsx**, **signature_pad** all installed; usage varies.
- **vite-plugin-pwa 1.2.0** — fully configured (manifest in `vite.config.ts:154-202`, service worker `client/public/sw.js`, runtime cache for `/api/trpc/*` as NetworkFirst, navigateFallbackDenylist for `/api/`).
- **vite-plugin-manus-runtime 0.0.57** — Manus-specific runtime injected into dev (must be removed for the rebuild).
- **@builder.io/vite-plugin-jsx-loc** — JSX source-location plugin.

### Database

- **MySQL** (production is **TiDB Cloud**, US-East-1 — confirmed by checked-in `.manus/db/*.json` query history, see Section 4). Driver: **mysql2 3.15.0**.
- **Drizzle ORM 0.44.5** + **drizzle-kit 0.31.4**. Schema at `drizzle/schema.ts` (~655 lines, 32 tables). Migrations in `drizzle/0000…0022_*.sql` (23 versioned migrations) plus `drizzle/meta/_journal.json`.
- DB client is lazy-singleton in `server/db.ts` — `getDb()` returns `null` if `DATABASE_URL` is unset, and **most call sites silently return `[]`/`undefined` on null DB**. There are no transactions wrapping multi-step writes (e.g. `applyMasterPackageToProject`, `seedProjectPricingOverrides`, `createSiteUpdate`+photos).
- Migration entry point: `pnpm db:push` runs `drizzle-kit generate && drizzle-kit migrate`. There is no rollback / down-migration tooling.

### File storage

- **Manus Forge storage proxy**, NOT direct S3 (despite `@aws-sdk/client-s3` and `@aws-sdk/s3-request-presigner` being in `package.json`, they are unused). All uploads/downloads go through HTTP calls to `BUILT_IN_FORGE_API_URL/v1/storage/upload?path=…` with `Authorization: Bearer ${BUILT_IN_FORGE_API_KEY}`. Implementation: `server/storage.ts` (`storagePut`, `storageGet`).
- Upload paths are namespaced by purpose, not tenant: `client-files/{projectId}/…`, `boq/{projectId}/…`, `{folder}/…` (admin generic uploader picks the folder). No tenant prefix today.
- The `aws-sdk` packages are dead dependencies and can be removed.

### Auth

Two unrelated auth systems coexist:

1. **Admin staff** — single shared username/password row in `admin_credentials` (`server/routers.ts:158-209`). Login compares bcrypt hash, issues a 7-day HS256 JWT in cookie `bm_admin_session`. Verified by `requireAdmin(ctx)` middleware (~85 callsites in `routers.ts`). No multi-user staff support; no roles.
2. **Manus OAuth** — `server/_core/oauth.ts` + `server/_core/sdk.ts`. Callback at `GET /api/oauth/callback` exchanges code with Manus' `webdev.v1.WebDevAuthPublicService`, upserts a `users` row, sets a 1-year HS256 JWT in cookie `app_session_id`. Used to populate `ctx.user` for tRPC procedures via `createContext`. The `OWNER_OPEN_ID` env var auto-promotes one user to `role: "admin"`. **In practice, OAuth-authenticated users only drive a few procedures** (e.g. chat sender name); the admin app uses the bcrypt path.
3. **Client portal** — no auth. Access is by URL-embedded `clientToken` (random nanoid, stored in `client_tokens`). Every portal procedure takes `{ token }`, calls `getClientTokenRecord(token)`, throws on missing/expired. No rate limit, no IP binding, no rotation.

JWT secret env var is `JWT_SECRET` for both systems, with a hard-coded fallback `"bmodern-fallback-secret-change-in-prod"` in `server/_core/jwt.ts:4` if unset.

### PDF generation

- **pdfkit 0.18.0** is what actually generates PDFs. Three documents:
  - Tender proposal — `server/pdf.ts` + `server/pdfRoute.ts` → `GET /api/pdf/proposal/:projectId` (admin) and `GET /api/pdf/portal/:token` (client).
  - Client selections summary — `server/clientSelectionsPdf.ts` → `GET /api/pdf/selections/:token`.
- **puppeteer 24.40.0**, **puppeteer-core 24.40.0**, **@sparticuz/chromium 147.0.0** are installed but unused. Dead dependencies; remove for the rebuild.
- PDFs are streamed inline (no storage). The PWA service worker is configured to bypass `/api/*` so download URLs work.

### Other Manus-platform integrations (server/_core/)

All wrap calls to `BUILT_IN_FORGE_API_URL` with the Forge API key:

- `dataApi.ts` — generic Manus DataService caller (not used by app code).
- `llm.ts` — LLM message types + `invokeLLM` (used by `boqRoute.ts` for AI BOQ extraction and by two `generateDescription`/`suggestWording` admin procedures in `routers.ts`).
- `notification.ts` — Manus push notification (used by the unique `system.notifyOwner` procedure).
- `imageGeneration.ts` — image generation helper (no callsites in the codebase outside the helper itself).
- `voiceTranscription.ts` — speech-to-text (no callsites).
- `map.ts` — Google Maps proxy (used by `client/src/components/Map.tsx`).
- `oauth.ts`, `sdk.ts`, `cookies.ts`, `jwt.ts`, `vite.ts`, `env.ts`, `context.ts`, `systemRouter.ts`, `types/manusTypes.ts` — server bootstrap, OAuth, session, dev-Vite middleware, env, tRPC context, system router.

### Testing

- **vitest 2.1.4** (`pnpm test` → `vitest run`). Config at `vitest.config.ts`. Test runner is Node, includes only `server/**/*.test.ts` / `*.spec.ts`. **No frontend tests.**
- 7 test files in `server/`:
  - `auth.logout.test.ts`, `chat-custom-items.test.ts`, `milestones-seed.test.ts`, `portal-features.test.ts`, `pricing-rules.test.ts`, `project-overrides.test.ts`, `tender-portal.test.ts`.
- No coverage reporting configured. No CI configuration anywhere in the repo (no `.github/workflows`, no `.gitlab-ci.yml`).

### Build, dev, and deployment

- Dev: `pnpm dev` → `NODE_ENV=development tsx watch server/_core/index.ts`. The Express server boots, then mounts Vite's middleware in dev (`server/_core/vite.ts`). Single port (default 3000, auto-finds next available up to 3019).
- Build: `pnpm build` → `vite build && esbuild server/_core/index.ts --platform=node --packages=external --bundle --format=esm --outdir=dist`. Client output: `dist/public/`. Server bundle: `dist/index.js`.
- Production: `pnpm start` → `NODE_ENV=production node dist/index.js`. Server serves `dist/public/` via `serveStatic`.
- **Deployment platform: Manus.** Confirmed by the live URL (`*.manus.space`), `vite-plugin-manus-runtime`, the `.manus/` and `client/public/__manus__/` directories, and `vite.config.ts:225` allowing `*.manus.computer` / `*.manuspre.computer` / `*.manus-asia.computer` / `*.manuscomputer.ai` / `*.manusvm.computer` hosts.
- **No infrastructure-as-code, no Dockerfile, no Procfile, no `fly.toml`, no `render.yaml`.** Deployment is opaque to the repo.

### Configuration / environment

All configuration is environment variables, loaded via `dotenv` on the server and `import.meta.env` on the client. Central server-side accessor: `server/_core/env.ts`.

| Variable | Required | Used by | Notes |
|---|---|---|---|
| `DATABASE_URL` | yes | `server/db.ts`, `drizzle.config.ts`, all seed scripts | MySQL connection string. `drizzle.config.ts` throws if missing; `db.ts` silently returns no DB. |
| `JWT_SECRET` | should be | `server/_core/jwt.ts`, `server/_core/sdk.ts` (`ENV.cookieSecret`) | Hard-coded fallback `"bmodern-fallback-secret-change-in-prod"` if unset — security risk. |
| `OAUTH_SERVER_URL` | yes (for OAuth) | `server/_core/sdk.ts` | Manus OAuth base URL. |
| `VITE_APP_ID` | yes (for OAuth) | `server/_core/env.ts`, `client/src/const.ts` | Manus app identifier. |
| `VITE_OAUTH_PORTAL_URL` | yes (for OAuth) | `client/src/const.ts` | Manus OAuth UI base URL. |
| `OWNER_OPEN_ID` | optional | `server/db.ts:78` | OpenID auto-promoted to role `admin` on user upsert. |
| `BUILT_IN_FORGE_API_URL` | yes (for storage/email/maps/LLM) | `server/storage.ts`, `server/email.ts`, `server/_core/{map,imageGeneration,voiceTranscription,llm,notification,dataApi}.ts` | Manus Forge proxy. |
| `BUILT_IN_FORGE_API_KEY` | yes (for storage/email/maps/LLM) | same as above | Bearer token for Forge proxy. |
| `VITE_FRONTEND_FORGE_API_URL` | optional | `client/src/components/Map.tsx` | Defaults to `https://forge.butterfly-effect.dev`. |
| `VITE_FRONTEND_FORGE_API_KEY` | optional | `client/src/components/Map.tsx` | Frontend-visible bearer (any user's browser sees this in network calls). |
| `NODE_ENV` | optional | `server/_core/index.ts`, `vite.config.ts` | Switches Vite middleware vs static serving. |
| `PORT` | optional | `server/_core/index.ts` | Default 3000; auto-increments if busy. |

There is **no `.env.example`** in the repo. New environments are set up by reading the code.
